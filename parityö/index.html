Olio-ohjelmointi

Olio-ohjelmointi on ohjelmointimalli, joka järjestää ohjelmistosuunnittelun datan tai objektien ympärille funktioiden ja logiikan sijaan. Objekti voidaan määritellä tietokenttään, jolla on ainutlaatuiset attribuutit ja käyttäytyminen.

Keskeiset sisäänrakennetut oliot

JavaScriptissä on neljä keskeistä sisäänrakennettua oliota, Array, Date, Math ja String. Jokaisella oliolla on siihen liittyviä erikoisominaisuuksia ja -menetelmiä.

Math

  • Math on sisäänrakennettu staattinen olio.
  • Sitä käytetään monimutkaisten matemaattisten operaatioiden suorittamiseen.

Date

  • Date manipuloi päivämäärää ja aikaa.
  • Date() konstruktori ei ota argumentteja.
  • Date olio antaa palauttaa ja asettaa vuodet, kuukaudet, päivät,tunnit, minuutit, sekuntit ja millisekuntit.

String

  • String käytetään tekstin kanssa työskentelemiseen.
  • Se toimii useiden merkkien kanssa.

Perusolion toteutus

class Cars {
constructor(name, maker, price) {
this.name = name;
this.maker = maker;
this.price = price;
}

getDetails(){
return (`The name of the car is ${this.name}.`)
}
}

let car1 = new Cars('Rolls Royce Ghost', 'Rolls Royce', '$315K');
let car2 = new Cars('Mercedes AMG One', 'Mercedes', '$2700K');

console.log(car1.name);
console.log(car2.maker);
console.log(car1.getDetails());

Konstruktori

Konstruktori on funktio, joka luo ja alustaa luokan objektiinstanssin. JavaScriptissä konstruktoria kutsutaan, kun objekti luodaan 'new' avainsanalla. Konstruktorin tarkoitus on luoda uusi objekti ja asettaa arvot olemassa oleville objektin ominaisuuksille.

class Person {
name;
constructor(name) {
this.name = name;
}

introduceSelf() {
console.log(`Hi! I'm ${this.name}`);
}
}

Kun konstruktori kutsutaan JavaScriptissä:

  • Uusi tyhjä objekti luodaan.
  • 'this' avainsana viittaa uuteen objektiin
  • Uusi objekti palautetaan sitten konstruktorin palautusarvona.

Kapselointi

Jokainen olion ominaisuus ja metodi on tallenettu olioon turvallisesti. Myös muiden olioiden metodit voivat vaikuttaa olioomme, jos annamme niille luvan. Jokaiselle oliolle voi määritellä julkisia ja yksityisiä muuttujia ja metodeita. Toiset oliot pääsevät käsiksi julkisiin muuttujiin, mutta yksityiset muuttujat ovat suojattuja. Tätä kutsutaan kapseloinniksi, ja se tekee koodista paremmin suojattua.

Esimerkki:

  • Sankarilla on usein yksityisiä muuttujia kuten nimi ja vaatteet, ja metodeita kuten hyökkäys, juoksu ja hyppy.
  • Sillä voi olla myös julkisia ominaisuuksia, kuten "elämät" ja metodeja kuten "vahingoittuminen", joihin muut oliot pääsevät käsiksi ja voivat jopa muuttaa niitä. "Vihollinen"-olio voisi vaikkapa hyökätessään kutsua funktiota "vahingoittuminen" ja muuttaa sen kautta sankarin "elämät" muuttujan arvoa.

class Student extends Person {

#year;

constructor(name, year) {
super(name);
this.#year = year;
}


introduceSelf() {
console.log(`Hi! I'm ${this.name}, and I'm in year ${this.#year}.`);
}

canStudyArchery() {
return this.#year > 1;
}
}

Periyttäminen

Kaiken takana on vain kolmenlaisia olioita: funktio-olioita, prototyyppiolioita ja "tavallisia" olioita. Kaikilla olioilla (paitsi yhdellä) on oma prototyyppinsä, josta se perii ominaisuuksia. Prototyyppiperiytymisen perustana ovat funktio-oliot Object ja Function. Jokaisella funktio-oliolla on oma prototyyppiolio, joka ei ole kuitenkaan funktion oma prototyyppi - paitsi yhdellä funktiolla: funktiolla Function prototyyppi ja prototyyppiolio ovat sama asia, sillä Function-olio perii funktio-ominaisuutensa omalta prototyyppiolioltaan. Konstruktorifunktioiden prototyyppiolioista muodostetaan linkitettyjä ketjuja, joista oliot perivät ominaisuuksia. Normaalisti jokaisen olion prototyyppien ketju päättyy Object-funktion prototyyppiolioon.

class Professor extends Person {

teaches;

constructor(name, teaches) {
super(name);
this.teaches = teaches;
}

introduceSelf() {
console.log(`My name is ${this.name}, and I will be your ${this.teaches} professor.`);
}

grade(paper) {
const grade = Math.floor(Math.random() * (5 - 1) + 1);
console.log(grade);
}
}
